# Report Format

Use this shape for the final output.

## 1. Idea Statement

- the idea in one sentence
- geography
- channel
- target customer if relevant
- matching note explaining what was actually tested

## 2. Claim Ledger

List each testable claim with its verdict.

Preferred shape:

```md
| Claim | Verdict | Confidence / 10 | Notes |
| --- | --- | --- | --- |
| There is visible public demand for X | supported | 7 | repeated category activity and buyer discussion |
```

## 3. Commercial Viability Snapshot

Include every row below. If no public evidence is available, set the readout to `unclear` and say what source would be needed.

```md
| Dimension | Readout | Confidence / 10 | Evidence or gap |
| --- | --- | --- | --- |
| Market existence / why this market exists |  |  |  |
| Buyer motivation / stable need |  |  |  |
| Differentiation |  |  |  |
| CAC proxy / acquisition pressure |  |  |  |
| LTV / repeat or retention logic |  |  |  |
| Content and SEO fit |  |  |  |
| Photo, video, UGC, and review fit |  |  |  |
| AI readability / citation / support fit |  |  |  |
| Logistics and fulfillment risk |  |  |  |
| Compliance and platform-policy risk |  |  |  |
| Margin / unit-economics plausibility |  |  |  |
| After-sales and support complexity |  |  |  |
| Brand compounding potential |  |  |  |
```

Use `supported`, `weakened`, or `unclear` style language in the readout. Do not fake exact CAC, LTV, margin, or return rates from public signals.

## 4. Coverage Summary

Provide a compact table or bullet list with:

- source type
- what it informed
- evidence quality / 10
- corroboration status
- notes

## 5. Strongest Supporting Facts

List the few facts that most strongly support the idea.

For each fact include:

- what it supports
- why it matters
- direct source links

## 6. Strongest Weakening Facts

List the few facts that most strongly weaken the idea.

For each fact include:

- what it weakens
- why it matters
- direct source links

## 7. Verdict

Use a compact commercial verdict.

Preferred shapes:

- `Promising but under-verified`
- `Demand-supported but crowded`
- `Interesting but channel-weakened`
- `Too unclear to validate from public evidence alone`

Then explain:

- what is actually supported
- what is actually weakened
- what remains unknown

## 8. Confidence And Gaps

State:

- stale or weak sources
- single-source or uncorroborated claims
- places where user misuse, shipping damage, seller-specific friction, or edge cases could explain a source
- unresolved claims
- claims that depend on private data
- where public evidence stops being enough
- private data needed for CAC, LTV, margin, supplier reality, returns, or support burden

## 9. Suggested Next Validation Step

Recommend one next step only, for example:

- test channel fit against three live listings
- interview five target buyers
- verify unit economics with supplier quotes
- run competitor and audience passes on the strongest rival
